## Рекомендації щодо покращення лендінгу

### A. Нові секції для додавання

#### 1. "Автоматизація фінансів" (нова Deep Dive секція)
**Контент:**
- Публічна сторінка оплати з QR-кодом (стандарт НБУ)
- 6 стратегій авто-матчингу платежів
- PDF-рахунки з QR-кодом для оплати
- Калькулятор собівартості з розбивкою витрат

**Чому важливо:** Це унікальна конкурентна перевага. Жодна інша ERP для 3D-друку не має автоматичного матчингу платежів з 6 стратегіями.

#### 2. "Розрахунок собівартості" (нова Features картка або Deep Dive)
**Контент:**
- Автоматичний розрахунок: філамент + електрика + знос принтера + праця + компоненти
- Рекомендована ціна за цільовою маржою
- Прибуток і маржа для кожного товару

**Чому важливо:** Ключова больова точка для 3D-друкарень -- вони часто не знають реальну собівартість виробів.

#### 3. "E-commerce інтеграція" (нова картка в Integrations або Features)
**Контент:**
- Вебхук для автоматичного прийому замовлень з інтернет-магазинів
- Автоматичне створення клієнтів
- Матчинг товарів по SKU
- Прив'язка e-com замовлень до платежів

#### 4. "Масштабування та команда" (нова секція або розширення Why Us)
**Контент:**
- Multi-tenant: кожна компанія -- ізольований простір
- 4 ролі: суперадмін, адмін, модератор, клієнт
- 16 модулів з гранулярними правами доступу
- Онбордінг-візард для нових співробітників

### B. Розширення існуючих секцій

#### Pain Points -- додати 2 картки:
5. **"Не знаєте реальну собівартість"** -- Електрика, знос принтера, філамент, праця -- скільки насправді коштує один виріб? Без точного розрахунку ви продаєте в мінус.
6. **"Клієнти питають 'куди платити?'"** -- Кожен раз шукаєте IBAN, формуєте призначення платежу, перевіряєте чи прийшли гроші вручну.

#### Features -- додати/замінити картки:
7. **Калькулятор собівартості** -- Автоматичний розрахунок вартості виробу: філамент, електрика, знос принтера, компоненти, праця.
8. **PDF та посилання для оплати** -- Генерація PDF-рахунків та публічних посилань з QR-кодом для оплати будь-яким банком.
9. **Клієнти** -- База клієнтів з історією замовлень, контактами, адресами НП/Укрпошти, VIP-статусом.
10. **E-commerce** -- Автоматичний прийом замовлень з інтернет-магазинів через вебхук.

#### Deep Dive -- додати блок:
4. **Собівартість та ціноутворення** -- Візуалізація розбивки собівартості: діаграма з сегментами (філамент, електрика, знос, праця, компоненти) + рекомендована ціна.

#### Integrations -- додати картку:
5. **E-commerce платформи** -- Prom.ua, Shopify, власний магазин -- автоматичне створення замовлень через вебхук.

### C. Текстові покращення

#### Hero subtitle -- поточний текст занадто загальний:
- **Зараз:** "Замовлення, черга виробництва, статуси друку та доставки в реальному часі."
- **Краще:** "Замовлення, собівартість, авто-матчинг платежів, Нова Пошта, Bambu Lab -- все в одній системі."

#### Trust line -- додати конкретику:
- **Зараз:** "Створено для українського бізнесу 3D-друку"
- **Краще:** "Monobank + Нова Пошта + Bambu Lab | Створено для українського 3D-друку"

## Пріоритетність змін

### Високий пріоритет (найбільший вплив на конверсію)
1. Додати **калькулятор собівартості** -- це головна больова точка і унікальна перевага
2. Додати **публічну сторінку оплати з QR** -- wow-ефект, економить час щодня
3. Розширити опис **авто-матчингу платежів** -- 6 стратегій це вражає
4. Додати **PDF-генерацію** -- практична щоденна потреба

### Середній пріоритет
5. Додати **e-commerce інтеграцію** -- для масштабування бізнесу
6. Розширити опис **системи статусів замовлень** (9 етапів)
7. Додати секцію **клієнти** -- окрема картка у Features
8. Додати **права доступу та ролі** -- для команд

### Низький пріоритет (nice to have)
9. Бандли товарів
10. Об'єднання замовлень
11. Постачальники
12. Онбордінг-візард

## Резюме

Поточний лендінг покриває ~40% реальних можливостей системи. Найбільші прогалини:

1. **Фінансовий блок** (калькулятор собівартості, QR-оплата, PDF) -- взагалі не представлений
2. **E-commerce інтеграція** -- не згадана, хоча це ключова точка масштабування
3. **Деталізація модулів** -- Features картки занадто абстрактні, не показують глибину функціоналу
4. **Команда та масштабування** -- ролі, права, multi-tenant не згадані

Рекомендую оновити лендінг в такому порядку: Pain Points (+2) → Features (+4) → Deep Dive (+1) → Integrations (+1) → Hero (текст).
